# Feature 需求：F3 - 环境自动上下文 (Auto Context)

## 1. 背景与目标
Agent 在新会话启动时，对整个项目的技术栈、入口点及核心逻辑缺乏“首屏感知”。用户往往需要花费额外的话语来描述背景。

**目标**：实现“开箱即用”，让 Agent 启动即具备对当前项目的全局视角。

## 2. 核心功能点

### 2.1 引导快照 (`boot_snapshot`)
- **功能**：启动时自动扫描根目录。
- **内容**：
  - 自动读取 `README.md` (项目说明)。
  - 自动解析 `package.json` 或 `go.mod` (技术栈)。
  - 自动生成 2 层深度的文件树预览。

### 2.2 上下文压缩
- **要求**：将扫描到的信息精简为 500 Token 以内的“上帝视角”摘要，注入 `MemoryManager` 的头部。

## 3. 验收标准
- [ ] 启动后，不输入任何指令，直接问“这个项目是做什么的”，Agent 能准确回答其核心技术栈。
- [ ] 启动感知过程应在 1s 内完成，且不占用过多的对话 Tokens。

## 4. 技术方案
- 在 `index.ts` 启动流程中增加 `ContextInformer` 模块。
- 预设基础文件的优先级权重。
