# Feature 需求：F1 - 源码全局感知 (Codebase Search)

## 1. 背景与目标
目前 CodeAgent 只能通过 `list_directory` 和 `read_file` 逐个探索文件，在处理中大型项目（>20个文件）时，Agent 难以快速定位相关的函数定义、外部引用或业务逻辑点。“盲人摸象”式的探索效率极低且容易遗漏上下文。

**目标**：赋予 Agent “扫描全局”的能力，使其能通过关键字、正则或符号定位，瞬间找到关键代码位置。

## 2. 核心功能点

### 2.1 语义/文本搜索 (`search_code`)
- **功能**：在整个工作区内进行文本搜索（类似于 `grep`）。
- **要求**：
  - 支持 `include/exclude` 文件过滤。
  - 返回匹配的文件路径、行号以及上下文片段。
  - 性能：对于 100 个文件以内的项目，响应时间应 < 500ms。

### 2.2 定义定位 (`find_definition`)
- **功能**：定位特定符号（类名、函数名）的定义位置。
- **技术路线**：
  - 初步：使用正则匹配常见的 `export class/function`。
  - 进阶：考虑接入简单的 AST 解析。

### 2.3 文件树拓扑感知 (`list_tree`)
- **功能**：一次性输出深度可控的文件树结构。
- **目的**：帮助 Agent 快速理解项目目录结构。

## 3. 验收标准
- [ ] Agent 能通过输入一个函数名，在不手动一个个读文件的情况下，说出它在哪个文件定义。
- [ ] 搜索结果自动受 `SecurityLayer` 保护，不得泄露工作区外的敏感信息。
- [ ] 具备 Token 保护：如果搜索结果过多，应自动截断并提示 Agent 缩小范围。

## 4. 技术选型建议
- 使用 Node.js 的 `fs.promises` 读取文件内容。
- 采用 `ripgrep` (如果环境支持) 或原生 JavaScript 正则进行匹配。
- 输出格式：标准 JSON 结构，方便 Agent 解析。
