# Feature 需求：F6 - 工作区授权 (Workspace Authorization / Trust Mode)

## 1. 背景与目标
CodeAgent 作为一个拥有强力执行权限（文件读写、命令执行）的工具，在任意目录启动即获得该目录的完整控制权。为了防止用户在敏感目录误启动 Agent 导致不可逆的修改，需要引入类似于 VS Code 的“信任工作区”机制。

**目标**：在 Agent 启动时，强制要求用户确认是否信任当前工作区并授权 Agent 进行操作。

## 2. 核心功能点

### 2.1 首次启动授权 (Trust Check)
- **功能**：当 Agent 在一个新目录启动时，提示用户授权。
- **UI 交互**：使用 `@inquirer/prompts` 弹出一个确认框。
- **确认文本**：`"检测到您在 [目录路径] 启动了 CodeAgent，您是否信任此工作区并授权 Agent 执行任务？"`

### 2.2 授权持久化
- **功能**：用户授权后，下次在该目录启动不再提示。
- **存储方案**：
  - 方案 A：在当前目录创建 `.codeagent/trust` 文件。
  - 方案 B：在全局配置目录（如 `~/.codeagent/trusted_workspaces.json`）记录绝对路径。
  - **建议**：优先使用方案 B，避免污染用户项目目录。

### 2.3 拒绝启动逻辑
- **功能**：如果用户选择“不信任”，Agent 应立即优雅退出，不开启任何 LLM 会话或工具初始化。

## 3. 验收标准
- [ ] 在从未运行过 CodeAgent 的目录启动，必须看到授权提示。
- [ ] 选择“Yes”后，Agent 正常进入交互模式。
- [ ] 退出并重新启动，不再出现提示。
- [ ] 选择“No”后，程序直接退出，且没有任何文件被修改。

## 4. 技术方案
- 在 `src/index.ts` 的入口逻辑中，位于 `engine` 初始化之前增加 `SecurityLayer.checkWorkspaceTrust()` 环节。
- 使用 `os.homedir()` 定位全局存储路径。
