# Feature 需求：F13+ - Slash 命令菜单增强 (Enhanced Slash Menu)

## 1. 背景与目标
目前 CodeAgent 的 Slash 命令通过输入 `/` 弹出选择菜单。为了提升用户在命令选择时的直觉和准确度，需要对该菜单进行分类管理，并提供实时的命令说明预览，降低用户的记忆负担。

## 2. 核心功能点

### 2.1 实时非阻塞建议 (Surgical Live Hints)
- **功能**：当用户在主输入框输入以 `/` 开头的字符时，不使用 `log-update` 全局刷新，而是通过 ANSI 转义序列手工在输入行下方的垂直区域绘制建议列表。
- **核心逻辑**：
    - **位置锁定**：记录当前光标位置，跳至输入框下一行进行清屏并绘制。
    - **非侵入式**：绘制完成后光标立即返回输入原点，确保输入框内容、提示符（╰─>）和当前输入字符不被破坏。
    - **实时响应**：每按下一个键都会触发一次增量更新，实现毫秒级响应。
- **界面效果**：
    - 每行：`╰─ 命令名称` [描述]
    - 支持前 5 条相关结果的滚动建议。

### 2.2 辅助交互菜单 (Optional Search Menu)
- **功能**：点击特定快捷键（或在需要时）仍可调用增强版分类菜单提供“向导式”选择。
- **状态栏提示**：在输入状态下，状态栏上方始终显示 `💡 建议:` 行，保持用户注意力集中。

## 3. 验收标准
- [ ] 输入 `/` 弹出的菜单中，命令按类别（Session, Model 等）分块显示。
- [ ] 在菜单中上下移动时，能够看到当前选中命令的详细功能介绍。
- [ ] 菜单风格与当前 CLI 整体视觉（Chalk 着色）保持一致。

## 4. 技术方案
- **数据结构**：在 `SlashCommandDef` 中增加 `category` 字段。
- **选型**：使用 `@inquirer/prompts` 的 `select` 组件，利用其 `description` 属性实现实时预览效果。
- **拦截逻辑**：在 `REPL` 类中通过 `TerminalManager.suspendInput` 挂起原始 readline 输入，待菜单选择完成后恢复。
