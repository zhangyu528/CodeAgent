# F13 交互式斜杠命令菜单 (F13_交互式命令菜单_实现计划) - [DONE]

## 1. 目标 (Objective)
当用户在 CLI 提示符开头输入 `/` 时，自动弹出一个交互式列表，展示所有可用的斜杠命令供用户选择，提升命令的发现性和输入效率。

## 2. 交互逻辑
1.  **触发**: 用户在 `CodeAgent > ` 提示符后，在输入任何内容前按下 `/` 键。
2.  **暂停**: CLI 自动调用 `ui.suspendInput` 暂停当前的 `readline` 会话。
3.  **显示**: 使用 `@inquirer/prompts` 的 `select` 组件展示所有已注册的斜杠命令及其描述。
4.  **选择**: 用户通过方向键选择命令并回车。
5.  **恢复**: 将选择的命令（如 `/model`）自动填入 `readline` 缓冲区，并恢复交互模式。

## 3. 技术方案

### 3.1 核心组件扩展 (`src/cli/keybindings.ts`)
*   重构 `attachKeybindings` 以支持可选的 `stdin` 输入流（提升可测试性）。
*   在 `onKeypress` 中监听字符 `/`。
*   逻辑判断：若当前模式为 `IDLE` 且 `readline` 当前行为空，则触发 `onSlash` 回调。

### 3.2 逻辑集成 (`src/index.ts`)
*   实现 `onSlash` 处理函数：
    *   调用 `ui.suspendInput` 以安全切换终端控制权给 `inquirer`。
    *   映射 `getDefaultSlashCommands()` 为 `choices` 列表。
    *   通过 `rl.write(selected)` 将用户选中的命令回填至输入行。

### 3.3 自动化测试
*   新增 `src/tests/unit/keybindings.test.ts`。
*   使用 Mock Stdin 模拟按键事件，验证在空行输入 `/` 时能正确触发回调，而在已有输入时则不触发。

## 4. 验收标准
1.  **自动弹出**: 在空白行输入 `/` 立即弹出菜单。
2.  **内容完整**: 菜单应包含所有斜杠命令及其简要描述。
3.  **回填功能**: 选择命令后，命令字符应正确出现在提示符后，且允许用户继续输入参数。
4.  **无误触**: 在输入路径或普通文本中包含 `/` 时不会触发菜单。
