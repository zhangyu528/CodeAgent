# 启动性能优化需求

## 1. 概述

### 1.1 背景

当前 Ink 程序启动时存在明显延迟，用户体验不佳。主要原因是启动过程中同步加载了过多数据，导致界面等待时间过长。

### 1.2 问题分析

| 问题点 | 当前行为 | 影响 |
|--------|----------|------|
| 历史记录加载 | 启动时同步加载 50 条 | 阻塞启动 |
| Agent 订阅 | 同步建立 | 可能有延迟 |
| **模型配置（已优化）** | **启动时加载 13896 行的 models.generated.js** | **主要阻塞原因** |

### 1.3 优化目标

- 启动时间减少 50%+
- 先显示 UI，后台加载数据
- 用户可立即进行交互

---

## 2. 现状分析

### 2.1 启动流程

```
PiInkApp 组件初始化
├── useReducer(piAppReducer, initialState)
├── useState(historyItems)           ← 初始为空
├── useState(currentSession)
├── useModelConfig(agent)           ← 导入 pi-ai 模块（13896行）
└── useEffect(() => {
      refreshHistory()  // ← 同步加载50条历史
    }, [refreshHistory])
```

### 2.2 问题根源

1. **pi-ai 模块导入**：13896 行的 `models.generated.js` 在 `import useModelConfig` 时同步加载
2. **历史记录加载**：启动时加载 50 条记录

---

## 3. 优化方案

### 阶段一：历史记录按需加载 ✅ 已完成

| 操作 | 优化前 | 优化后 |
|------|--------|--------|
| 启动时 | 加载 50 条 | 只加载 1 条（最新会话 ID） |
| /history 命令 | 使用已加载数据 | 触发时再加载完整列表 |
| /resume 命令 | 需要等待 | 即时可用（已有最新会话 ID） |

**具体改动：**

1. `refreshHistory(limit?: number)` 添加可选参数
2. 启动时调用 `refreshHistory(1)` 只获取最新会话
3. `/history` 命令时调用 `refreshHistory(50)` 获取完整列表

### 阶段二：初始化状态管理 ✅ 已完成

| 状态 | 描述 |
|------|------|
| `isInitializing` | 是否正在初始化 |
| `isFirstHistoryLoadRef` | 追踪首次历史记录加载 |

**UI 表现：**
- 初始化期间：显示 "Loading..." 提示
- 初始化完成后：正常显示 Welcome 页面

### 阶段三：模型配置延迟加载 ✅ 已完成

**问题根源：**
- `useModelConfig.ts` 在模块顶层执行 `import { getProviders, getModels } from '@mariozechner/pi-ai'`
- 这会同步加载 13896 行的 `models.generated.js`

**解决方案：**
- 移除静态 import，改用动态 `import()` 懒加载
- 只有当用户触发 `/model` 命令时才加载模型列表
- 添加 `isLoading` 状态追踪加载进度

**改动：**
```typescript
// 之前：静态导入（启动时加载）
import { getModels, getProviders } from '@mariozechner/pi-ai';
const PROVIDERS = getProviders().filter(...);

// 之后：动态导入（需要时加载）
async function ensureProvidersLoaded(): Promise<string[]> {
  if (providersCache) return providersCache;
  const { getProviders: gp, getModels: gm } = await import('@mariozechner/pi-ai');
  // ...
}
```

## 4. 优化后启动流程

```
PiInkApp 组件初始化
├── useReducer(piAppReducer, initialState)
├── useState(historyItems)           ← 初始为空
├── useState(currentSession)
├── useModelConfig(agent)           ← 不再导入 pi-ai 模块 ✅
├── isInitializing = true           ← 显示 Loading... ✅
└── useEffect(() => {
      refreshHistory(1)             ← 只加载1条历史 ✅
    }, [refreshHistory])
```

**启动后：**
1. 显示 "Loading..."
2. 后台加载 1 条历史记录
3. 初始化完成，显示 Welcome 页面
4. 用户可以立即交互

**用户触发 /model 时：**
1. 显示选择 Provider 界面
2. 动态加载 pi-ai 模型列表
3. 显示 Model 选择列表
