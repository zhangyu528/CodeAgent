# 功能需求 N5：会话治理与检索增强

## 背景与问题
在 N3/N4 打好基线后，session 仍缺乏治理与检索能力，难以支持长期使用：
- 会话数量增长后缺少归档、删除、保留策略。
- 难以通过标题快速定位历史上下文。
- 缺乏 session 级 token/cost 聚合，不利于成本管理。

## 目标
- 提供基础会话治理能力：归档、删除、导出/导入、TTL 清理。
- 提供可用检索能力：标题/关键词筛选与最近优先排序。
- 增加 session 级观测指标：token、cost、最后活跃时间。

## 非目标
- 本阶段不做跨项目全局搜索排名。
- 本阶段不做向量数据库与语义召回。
- 本阶段不做远程云同步。

## 设计要点
### 治理能力
- `archive(sessionId)`：从默认列表隐藏但可恢复。
- `delete(sessionId)`：软删除优先，支持恢复窗口。
- `export/import`：JSON bundle，保证跨环境迁移。
- `retention policy`：支持按数量和按时间双策略清理。

### 检索能力
- `listRecent` 增加过滤参数：`query/status/provider/model`。
- 默认排序：`updatedAt DESC`。
- CLI 侧支持 history 内快速过滤与高亮匹配。

### 观测能力
- session 粒度聚合字段：
  - `totalInputTokens`
  - `totalOutputTokens`
  - `totalCost`
  - `lastTool`
  - `lastError`
- 与 UI 状态栏联动，支持查看当前 session 消耗摘要。

## 用户可见行为
- 用户可在历史中归档/删除会话，减少列表噪音。
- 用户可通过关键词快速定位目标会话。
- 用户可看到会话累计 token/cost，辅助控制开销。

## 验收标准
- Given 历史会话超过阈值，When 执行清理策略，Then 仅清理符合规则会话且可审计。
- Given 用户输入关键词，When 在历史中筛选，Then 返回结果与标题/摘要匹配一致。
- Given 多轮会话，When 查看统计，Then token/cost 聚合与消息 usage 累计一致。

## 风险与回滚
- 风险：删除策略误删重要会话。
- 缓解：软删除+恢复窗口+导出备份。
- 回滚：可关闭治理策略，仅保留只读历史列表。

## 里程碑
- M1：归档/删除/导出导入接口与 CLI 操作入口。
- M2：history 过滤检索与 UI 交互增强。
- M3：session 级 token/cost 聚合与可视化展示。
