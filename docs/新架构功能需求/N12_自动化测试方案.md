# N12: Ink TUI 自动化测试方案

本文档记录 CodeAgent Ink TUI 的自动化测试策略与实施计划。

## 背景

随着 Ink TUI 功能的复杂度增长，需要建立可靠的自动化测试体系，确保：
- Slash 命令执行逻辑正确
- 消息处理流程可靠
- 页面切换与状态管理正确
- 回归测试覆盖关键功能

## 2. 组件测试 (Component Tests)

### 方案

使用 Ink 内置的 `renderToString` 进行输出验证。

```typescript
import { renderToString } from 'ink';
import { SlashList } from '../../src/apps/cli/ink/components/inputs/SlashList.js';

describe('SlashList Component', () => {
  it('should render slash commands', () => {
    const output = renderToString(
      <SlashList
        inputValue="/"
        setInputValue={() => {}}
      />
    );
    expect(output).toContain('/help');
    expect(output).toContain('/new');
    expect(output).toContain('/model');
  });

  it('should filter commands based on input', () => {
    const output = renderToString(
      <SlashList
        inputValue="/m"
        setInputValue={() => {}}
      />
    );
    expect(output).toContain('/model');
    expect(output).not.toContain('/help');
  });
});
```

### Ink 组件测试限制

- Ink 是 TUI 组件库，`renderToString` 输出 ANSI 转义序列
- 需要解析 ANSI 序列进行断言
- 对于复杂交互（键盘事件），建议使用 mock `useInput`

## 3. 集成测试 (Integration Tests)

### Mock `useInput` 方案

TUI 的核心是键盘输入事件。测试需要 mock `ink` 的 `useInput`。

```typescript
import { describe, it, expect, vi } from 'vitest';

// Mock useInput
vi.mock('ink', async () => {
  const actual = await vi.importActual('ink');
  return {
    ...actual,
    useInput: (handler: (input: string, key: Key) => void) => {
      // 存储 handler 供测试调用
      window.__inputHandler = handler;
    },
    useApp: () => ({ exit: vi.fn() }),
  };
});

describe('Slash Command Flow', () => {
  it('should execute /help when Enter is pressed with /help input', async () => {
    const { getInputHandler } = await import('./testUtils.js');

    // 模拟输入
    const handler = getInputHandler();
    handler('/', {});
    handler('h', {});
    handler('\r', { return: true });

    // 验证 openNotice 被调用
  });
});
```

### 测试场景

| 场景 | 输入 | 预期结果 |
|------|------|---------|
| slash 命令执行 | `/h` + Enter | 显示 `/help` 命令帮助 |
| 前缀匹配 | `/m` + Enter | 执行 `/model` |
| 未知命令 | `/xyz` + Enter | 无操作或提示 |
| 消息发送 | `hello` + Enter | 消息显示并发送给 agent |
| 页面切换 | `hello` + Enter (from welcome) | 切换到 chat 页面 |

## 4. E2E 测试（未来方向）

对于完整的 TUI E2E 测试，可考虑：

| 工具 | 特点 |
|------|------|
| `ttytest` | 专为 TUI 设计，模拟终端输入 |
| `node-pty` | 伪终端，支持 fork 进程 |
| `@inkjs/testing-library` | Ink 官方测试工具（规划中） |

### 示例（伪代码）

```typescript
import { spawn } from 'child_process';

it('full slash command flow', async () => {
  const cli = spawn('bun', ['run', 'src/apps/cli/index.tsx']);

  // 等待启动
  await waitFor('Model:');

  // 输入 /help
  cli.stdin.write('/help\r');

  // 验证输出
  const output = await collectOutput(cli);
  expect(output).toContain('Available Commands');
  expect(output).toContain('/new');
  expect(output).toContain('/model');

  cli.kill();
});
```

```yaml

name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install
      - run: bun run test:run
```
