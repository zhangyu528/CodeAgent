# N12: Ink TUI 自动化测试方案 - 实施计划

## Context

当前项目已建立基础测试框架（Vitest + 1个单元测试），但覆盖率极低。核心功能如会话管理、消息处理、slash命令执行等均无测试覆盖。需建立完整的测试金字塔，保证功能完整度和质量。

## 当前现状

| 测试类型 | 状态 | 文件 |
|---------|------|------|
| 单元测试 | 仅1个 | `tests/unit/executeSlash.test.ts` (23用例) |
| 集成测试 | 无 | - |
| 组件测试 | 无 | - |

## 测试分层策略

```
┌─────────────────────────────────────────┐
│           E2E 测试 (未来)                │

├─────────────────────────────────────────┤

└─────────────────────────────────────────┘
```

## 第一阶段：单元测试扩展

### 1.1 useAppSession 单元测试

**文件**: `tests/unit/useAppSession.test.ts`

**测试用例**:

| 用例 | 描述 |
|------|------|
| `createSessionId` | 生成唯一 ID |
| `extractSessionTitle` | 标题提取逻辑（空输入、过长截断） |
| `ensureSessionForPrompt` | 新会话创建、已有会话复用 |
| `setPendingPrompt` | 设置并获取 pending prompt |
| `getAndClearPendingPrompt` | 获取后清空 |

**依赖 mock**:
- `getAgent()` → mock Agent 实例
- `sessionManager` → mock SessionManager

**依赖 mock**:
