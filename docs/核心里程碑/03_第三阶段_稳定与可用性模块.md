## 第三阶段：稳定与可用性模块 (产品化阶段)

在拥有了强大的规划和执行能力后，第三阶段的目标是将其打造成一款对用户来说安全、可信且友善的成熟终端产品。本阶段的开发任务对应于规划的 **P2 稳定性模块** 与 **P3 可用性模块**。

### 包含的开发模块 (对应 P2、P3 开发优先级)

- **Memory (记忆与上下文)** (P2)
  - 实现基于 Token 严格控制的短期滑动窗口架构，避免超出 LLM 最大输入限制。
  - 建立长期任务状态序列化落盘方案（如基于轻量级 JSON/SQLite），使其支持会话中断恢复。

- **Security Layer (安全层)** (P2)
  - **(基础实现已在 P1 预接入)**：已实现 Workspace 路径隔离校验与高危命令黑名单。
  - P2 重点：实现 HITL (Human-in-the-loop)，遇到高敏感操作主动挂起并请求用户命令行确认。

- **Observability (可观测性与日志)** (P3)
  - 接入 `pino`/`winston` 等成熟 Node.js 日志库做多等级监控。
  - 分离后台推理耗时、API Token 开支记录与业务执行信息，为排错和算账能力打好基础。

- **CLI Interface (命令行交互层)** (P3)
  - 利用 `commander` 和 `@inquirer/prompts` 重构系统的入口，提供现代化的 CLI 交互。
  - 将复杂的底层状态以优雅的方式渲染至前端 (终端)，包含动画 Spinner 与进度指引。

### 系统能力与验收标准

系统在此阶段应具备以下完整能力：

-   **稳定运行**：不轻易发生上下文爆炸导致的内存溢出或 API 报错。
-   **安全受控**：拒绝破坏性命令，保证不对当前工作区外的系统环境造成破坏。
-   **可观测**：生成并可实时跟踪 Agent 幕后的“推理和试错过程”。
-   **可交互**：成为真正的终端工具程序 (如执行 `agent run "重构代码"` 即可开启极佳的使用体验)。
