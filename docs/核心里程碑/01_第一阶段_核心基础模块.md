## 第一阶段：Agent Core (MVP 基础版)

本阶段的目标是建立 Coding Agent 的“骨架”，即能跑通大模型调用并执行本地工具的具体回路。整个第一阶段完全基于 **Node.js + TypeScript** 进行构建。

### 包含的开发模块 (对应 P0 开发优先级)

第一阶段的核心开发完全对应于 **P0 核心基础模块** 中规划的内容，具体包含以下三大核心模块：

- **LLM Engine (大模型引擎)**
  - 采用 **Provider 模式** 架构，建立统一的 `LLMProvider` 和 `LLMEngine` 接口。
  - 完成至少一个 LLM Provider (例如 OpenAI Provider 或 Anthropic Provider) 的接入。
  - 支持普通消息对话、流式输出 (Streaming) 以及解析工具调用请求。

- **Tool System (工具系统)**
  - 基于 `zod` 库建立标准化的 TypeScript 工具接口。
  - 完成核心架构，能将 TypeScript 函数的参数定义自动转为 LLM 的 Function Calling Schema。
  - 实现少数基本工具 (如读取文件 `ReadFileTool`、简单命令执行 `RunCommandTool` 等) 作为 MVP 验证。

- **Agent Controller (流程控制器)**
  - 实现核心的 Agent 执行循环 (Agent Loop)。
  - 跑通简单的 `LLM -> 解释意图 -> Tool -> 拿到结果 -> 返回给 LLM` 闭环。

### 系统能力与验收标准

系统在此阶段应具备以下最小能力：

**LLM → Tool → Result**

可以通过编写一个简单的测试脚本或入口文件，向 Agent 输入自然语言指令（例如：“读取 index.ts 的前 10 行并总结”），系统能够：
1. 正确调用大模型并将其意图转化为读文件工具的调用。
2. 在本地正确读取该文件。
3. 将读取结果交回大模型，并由大模型最终输出总结文本。
