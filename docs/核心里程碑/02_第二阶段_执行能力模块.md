## 第二阶段：执行能力模块 (Agent Loop 完整版)

在第一阶段跑通 LLM 与工具调用的基础上，第二阶段的目标是赋予 Agent 对物理世界产生实质影响的能力，并具备基本的复杂任务规划分解能力。此阶段核心开发对象对应 **P1 执行能力模块**。

### 包含的开发模块 (对应 P1 开发优先级)

- **Execution Environment (执行环境)**
  - 基于 Node.js `child_process` (及/或 `zx`) 建立稳定的底层代码/脚本/命令执行层。
  - 能够正确捕获命令的输出流 (`stdout`/`stderr`) 及退出码，并实现超时熔断机制。
  - 为未来的安全层接入预留中间件或拦截器位置。

- **Planner (任务规划器)**
  - 引入长期任务分解的能力，告别简单的单步骤 ReAct。
  - 通过大语言模型优先构建结构化执行计划 (例如分析依赖关系并产出 DAG 图或任务列表)。
  - 调度底层第一阶段的 Agent Controller 完成逐步执行。

### 系统能力与验收标准

系统在此阶段应具备以下增强能力：

**User Task → LLM → Plan → Tool (Execution) → Result**

可以通过编写一个中等复杂度的测试入口指示 Agent 完成连贯性任务（例如：“跑通项目单元测试，如果报错帮我分析并修复第一个错误”），系统能够：
1. 分析指令，列出明显的步骤 (1. 运行测试 2. 获取输出分析 3. 可能的修改操作)。
2. 通过 Execution Environment 实际运行终端中的 `npm test` 或相近命令。
3. 读入测试结果后，若存在失败情况，调用 Controller 执行文件分析与修改工具。
4. 提供连贯的长步骤体验，证明 Agent 具备“做事并拿到结果”的能力。
