## 1. Security Layer (安全层实现)

安全层是 CodeAgent 的“护城河”，负责拦截所有非法的路径访问和高危命令执行。

### 1.1 核心特性
- **集中式路径审计 (Path Guard)**：通过 `SecurityLayer.validatePath()` 统一校验，防止 Agent 读写工作区 (`D:\work\project\CodeAgent`) 之外的任何文件。
- **命令审计 (Command Auditing)**：实时扫描 `run_command` 参数，匹配内置的 Blocklist (如 `rm -rf /`)。
- **人机领航 (HITL)**：
  - 对敏感命令（如安装依赖、删除文件、Git 操作）开启交互式确认。
  - 用户必须在终端输入 `y/yes` 才能继续执行，否则 Agent 会收到拒绝提示。

### 1.2 关键类与方法 (`src/controller/security_layer.ts`)
```typescript
export class SecurityLayer {
  // 校验路径是否在工作区内
  validatePath(targetPath: string): boolean;
  
  // 检查命令安全等级 (SAFE, BLOCKED, SENSITIVE)
  checkCommand(command: string): { status: CheckStatus; reason?: string };
  
  // 触发用户审批 (HITL)
  async requestApproval(description: string): Promise<boolean>;
}
```

## 2. Memory Management (智能记忆管理)

记忆管理解决了 LLM 上下文溢出的物理限制，同时平衡了运行成本。

### 2.1 核心特性
- **Token 感知滑动窗口 (Sliding Window)**：基于字符估算（~4字符/Token），将上下文严格限制在 **4000 Tokens** 以内。
- **系统提示词锁 (System Prompt Lock)**：确保核心角色设定永远不会被截断算法丢弃。
- **透明可观测性**：在 CLI 每一轮交互末尾展示当前 Context 的 Token 消耗看板。

### 2.2 关键类与方法 (`src/controller/memory_manager.ts`)
```typescript
export class MemoryManager {
  // 写入新消息并自动触发滑动窗口裁剪
  addMessage(msg: Message): void;
  
  // 获取当前所有受控的记忆消息
  getMessages(): Message[];
  
  // 获取当前 Token 估算值
  getUsage(): number;
}
```
