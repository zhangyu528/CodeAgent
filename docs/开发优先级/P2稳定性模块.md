# P2：稳定性模块详细设计 (Node.js + TypeScript)

在 Agent 获得了执行任务的能力 (P1) 后，为了使其能够在真实的、非受限的项目环境中稳定且安全地长时间运行，P2 阶段的核心是引入 Memory (记忆/上下文) 与 Security Layer (安全层)。

## 1. Memory (记忆与上下文管理)

受限于 LLM 的 Token 上限和高昂的 API 调用成本，Agent 不能无限制地将所有历史记录丢给模型。Memory 模块负责高效地管理短期历史和长期知识。

- **核心职责**：
  - **短期记忆 (Short-term Memory)**：管理当前 Agent Loop 中的对话历史 (Message History)，实施滑动窗口 (Sliding Window) 或基于 Token 估算的自动截断 (Truncation) 与摘要 (Summarization)。
  - **长期记忆 (Long-term Memory)**：保存跨会话的任务状态、已学习的项目偏好或知识库。
  - **状态持久化**：将发生中断或退出的 Agent 状态落盘，支持从中断点恢复执行。

- **技术选型**：
  - **Token 计算**：使用 `tiktoken` (针对 OpenAI 模型) 或提供商对应的 Tokenizer 库在 Node.js 侧精准计算 Token 开销。
  - **持久化存储**：
    - 短期：直接内存管理或序列化为基于本地的 JSON/SQLite 文件 (`better-sqlite3`)。
    - 长期/向量：可考虑使用轻量级的本地向量库如 `chromadb` 或 `faiss-node` 来基于语义检索历史经验。
  - **接口定义 (TypeScript)**：
    ```typescript
    export interface MemoryItem {
      role: 'user' | 'assistant' | 'system' | 'tool';
      content: string;
      tokenCount?: number;
    }

    export class MemoryManager {
      private history: MemoryItem[] = [];
      private maxTokens: number;

      // 添加新记忆并自动根据 Token 上限截断或触发摘要
      add(item: MemoryItem): void { ... }
      
      // 获取用于本次 LLM 调用的记忆快照
      getSnapshot(): MemoryItem[] { ... }
      
      // 持久化到本地 SQLite 或 JSON
      async saveState(sessionId: string): Promise<void> { ... }
      async loadState(sessionId: string): Promise<void> { ... }
    }
    ```

## 2. Security Layer (安全层)

Coding Agent 拥有直接执行 Shell 命令和读写文件的能力，这带来了不可忽视的安全风险 (例如：误删文件 `rm -rf /`，泄露环境变量等)。

- **核心职责**：
  - **Workspace 沙箱 (Jailbreak 预防)**：限制 Agent 的所有文件读写操作只能在通过验证的 `cwd` (当前工作目录) 内发生，防止路径穿越 (`../../`)。
  - **Command Allowlist/Blocklist**：基于设定的规则阻断高危命令。
  - **Human-in-the-Loop (HITL)**：在执行被判定为“敏感”的操作或涉及高风险更改前，挂起流程并强制要求用户在 CLI 授权确认。

- **技术选型**：
  - **路径校验**：通过 Node.js 的 `path.resolve` 和 `path.relative` 原生方法严格校验解析后的绝对路径是否位于合法工作区前缀内。
  - **命令审计**：在 `ExecutionEnvironment` (P1) 调用 `child_process` 之前插入中间件 (Middleware) 或拦截器 (Interceptor)。
  - **接口定义 (TypeScript)**：
    ```typescript
    export class SecurityLayer {
      private allowedWorkspace: string;
      private blockedCommands: string[] = ['rm -rf', 'format', 'mkfs'];

      constructor(workspacePath: string) {
        this.allowedWorkspace = path.resolve(workspacePath);
      }

      // 验证文件路径是否安全
      validatePath(targetPath: string): boolean {
        const resolved = path.resolve(this.allowedWorkspace, targetPath);
        return resolved.startsWith(this.allowedWorkspace);
      }

      // 验证命令是否安全
      validateCommand(command: string): { isSafe: boolean; reason?: string } { ... }

      // 触发用户确认 (通过回调接入 CLI)
      async requestUserApproval(actionDescription: string): Promise<boolean> { ... }
    }
    ```

## 总结

P2 的 Memory 赋予了 Agent 的“续航”和归纳能力，而 Security Layer 则为其加上了强制的“安全带”。两者结合，使得由 Node.js 驱动的本地 Coding Agent 从一个玩具式的脚本，蜕变为可以被信任和长期托管的开发助手。
