# P3：可用性模块详细设计 (Node.js + TypeScript)

在系统的核心能力 (P0)、执行能力 (P1) 以及稳定性与安全保障 (P2) 皆开发完毕后，系统已经具备了完整且健壮的处理逻辑。P3 阶段的目标是将这一强大的内核包装为用户友好的终端产品，涵盖日志观测 (Observability) 与极佳打磨的命令行交互界面 (CLI Interface)。

## 1. Observability (可观测性与日志)

由于 Agent 在后台执行推导和工具调用时会有大量的中间状态，开发者/用户需要非常清晰的视图来了解“Agent 到底在想什么、做什么、为什么卡住？”。

- **核心职责**：
  - **分级日志**：区分 Debug (内部推理数据、Raw Prompt)、Info (常规状态如“正在思考”，“正在执行命令”)、Warn (如部分执行失败并触发重试)、Error (致命异常)。
  - **执行看板**：实时输出 Agent 的思考链路（Thought、Action、Observation），实现类似终端进度条的效果。
  - **成本追踪 (Telemetry)**：统计每个 Session 的 Token 消耗与预估 API 账单。

- **技术选型**：
  - **日志库**：推荐使用成熟的高性能 Node.js 日志库 `pino` 或通用的 `winston`，支持将格式化数据输出到控制台或持久化写入本地日志文件 (如 `.coder-agent/logs/`)。
  - **终端动画**：采用 `ora` (终端 Spinner) 显示动态加载状态，使用 `chalk` 输出鲜明的终端彩色高亮文本区分关键信息。
  - **接口定义 (TypeScript)**：
    ```typescript
    export interface Logger {
      debug(message: string, meta?: any): void;
      info(message: string, meta?: any): void;
      warn(message: string, meta?: any): void;
      error(message: string | Error, meta?: any): void;
    }

    export class TelemetryMonitor {
      // 记录调用耗费并聚合
      recordTokens(model: string, inputTokens: number, outputTokens: number): void { ... }
      getEstimatedCost(): number { ... }
    }
    ```

## 2. CLI Interface (命令行交互层)

CLI 是普通用户接触 Coding Agent 的第一门户。不仅需要承接用户输入，还要完美对接底层的 Controller 和 Security Layer。

- **核心职责**：
  - **命令路由**：解析传入的启动命令 (如 `agent run <task>`, `agent chat`)。**（基础版已在 P1 实现）**
  - **交互式问答**：在 Security Layer 触发拦截时，或需要用户确认时，通过漂亮终端 UI 发起确认。
  - **环境感知**：**（已在 P1 通过 SystemInfoTool 预实现）** 帮助 LLM 识别操作系统。
  - **配置管理**：管理 API Key 注入、选择大语言模型 Provider 及修改底层参数。

- **技术选型**：
  - **命令解析框架**：`commander` 或 `yargs`，提供声明式的命令行选项支持与帮助文档自动生成。
  - **交互式 UI**：`inquirer.js` 或其更现代的后继者 `@inquirer/prompts`，用于构建漂亮的单选、多选列表和安全密码输入框。
  - **整体架构 (入口点)**：
    ```typescript
    import { Command } from 'commander';
    import { confirm } from '@inquirer/prompts';
    
    const program = new Command();

    program
      .command('run <taskDescription>')
      .description('针对当前项目执行特定开发任务')
      .action(async (taskDescription) => {
        // 1. 初始化依赖注入集合 (DI)
        // 2. 组装 LLMEngine, ExecutionEnvironment, SecurityLayer 等
        // 3. 注册从 SecurityLayer 发出的 requireApproval 钩子绑定到 inquirer 的 confirm()
        // 4. 将 taskDescription 送入 Controller 启动流程
      });

    program.parse(process.argv);
    ```

## 总结

P3 阶段并非点缀，而是决定项目最终留存率的关键一环。现代化的 TypeScript CLI 配合 `ora`、`chalk` 及 `inquirer` 等库，可以构建比肩原生应用的丝滑体验。结合缜密的 Observability 链路记录，开发者可以在黑盒的生成式 AI 与追求确定性的本地编码环境之间找到极佳的平衡点。
