# F13 交互式斜杠命令菜单 (F13_交互式命令菜单_实现计划) - [DONE]

## 1. 目标 (Objective)
当用户在 CLI 提示符开头输入 `/` 时，自动弹出一个交互式浮层，展示所有可用的斜杠命令供用户选择。在 Blessed TUI 架构下，此功能通过 `SlashCommandPopup` 实现。

## 2. 交互逻辑
1.  **触发**: 用户在输入框输入 `/` 字符时。
2.  **显示**: 在输入框上方自动弹出命令列表浮层。
3.  **过滤**: 随着用户继续输入，列表实时过滤匹配的命令。
4.  **选择**: 用户通过 `Up/Down` 方向键在浮层中移动选择。
5.  **补全**: 按下 `Tab` 或 `Enter` 键，选中的命令将自动补全到输入框中。
6.  **关闭**: 按下 `Esc` 键或清空输入后浮层自动消失。

## 3. 技术方案

### 3.1 核心组件 (`src/apps/cli/components/slash_popup.ts`)
*   使用 `blessed.list` 创建悬浮层。
*   监听 `InputManager` 的输入事件，实时更新列表内容。

### 3.2 逻辑集成 (`src/apps/cli/components/input_manager.ts`)
*   在 `inputText` 的 `keypress` 事件中捕获输入并通知 `popup`。
*   处理快捷键劫持（`up/down/tab/enter/escape`）以实现交互。

### 3.3 自动化测试
*   新增 `src/tests/unit/keybindings.test.ts`。
*   使用 Mock Stdin 模拟按键事件，验证在空行输入 `/` 时能正确触发回调，而在已有输入时则不触发。

## 4. 验收标准
1.  **自动弹出**: 在空白行输入 `/` 立即弹出菜单。
2.  **内容完整**: 菜单应包含所有斜杠命令及其简要描述。
3.  **回填功能**: 选择命令后，命令字符应正确出现在提示符后，且允许用户继续输入参数。
4.  **无误触**: 在输入路径或普通文本中包含 `/` 时不会触发菜单。
