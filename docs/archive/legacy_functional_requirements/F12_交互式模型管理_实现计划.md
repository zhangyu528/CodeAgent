# F12 交互式模型与 Provider 管理 (F12_交互式模型管理_实现计划)

## 1. 目标 (Objective)
为 CLI 引入 **模型 (Model)** 和 **服务商 (Provider)** 的 **交互式选择与切换** 能力。

## 2. 核心功能

### 2.1 交互式模型选择 (/model)
*   **命令**: `/model` (无参数)
*   **交互**:
    *   调用当前 Provider 的 `listModels()` API 获取可用模型列表。
    *   弹出交互式列表 (`Select`)，列出所有模型。
    *   用户选择后，即时切换当前会话的模型配置，并在界面上反馈。
    *   **Fallback**: 若 API 不支持列出或失败，提示用户手动输入模型名称。

### 2.2 交互式 Provider 切换 (/provider)
*   **命令**: `/provider` (无参数)
*   **交互**:
    *   调用 `LLMEngine.listProviders()` 获取已注册的 Provider 列表 (如 `deepseek`, `glm`, `openai`)。
    *   弹出交互式列表 (`Select`)。
    *   用户选择后，切换全局默认 Provider，并自动加载该 Provider 的默认模型。
    *   状态栏即时更新显示新的 Provider/Model。

### 2.3 初始化向导增强 (Wizard Enhancement)
*   在 `verifyConnection` 成功后，自动调用 `listModels()`。
*   **若获取成功**:
    *   弹出模型选择列表 (推荐 Code 模型置顶)。
    *   用户选择的模型将被写入 `.env` 作为默认模型。
*   **若获取失败**:
    *   保留原有逻辑 (手动输入或默认值)。

## 3. 技术方案

### 3.1 LLMProvider 接口升级
在 `src/llm/provider.ts` 中新增可选方法：
```typescript
interface LLMProvider {
  // ... existing methods
  listModels?(): Promise<string[]>; 
}
```

### 3.2 Provider 实现
*   **OpenAI / DeepSeek / GLM / Ollama**:
    *   实现 `listModels()`：调用 `GET /v1/models` (或对应厂商端点)。
    *   解析响应 JSON，提取 `id` 字段列表。
*   **Anthropic**:
    *   实现 `listModels()`：返回静态推荐列表 (如 `claude-3-5-sonnet-latest`, `claude-3-opus-latest`, `claude-3-haiku-20240307`)，因为官方 API 暂不支持动态列出。

### 3.3 Slash Command 实现 (`src/cli/slash_commands.ts`)
*   **`/model` Handler**:
    *   获取当前 Provider。
    *   `await provider.listModels()`。
    *   调用 `@inquirer/prompts` 的 `select`。
    *   更新 Provider 实例的 `defaultModel` (需在 Provider 类中暴露 setter 或 `setModel` 方法)。
*   **`/provider` Handler**:
    *   获取 `engine.listProviders()`。
    *   调用 `@inquirer/prompts` 的 `select`。
    *   调用 `controller.switchProvider(name)`。

### 3.4 状态栏更新
*   修改 `HUD` 渲染逻辑，在 `Provider` 字段旁增加 `Model` 显示 (例如 `Provider: deepseek (deepseek-chat)`).

## 4. 验收标准
1.  **`/model`**: 输入后能看到当前 Provider 的模型列表，选择后即时生效。
2.  **`/provider`**: 输入后能看到已配置的 Provider 列表，选择后切换成功。
3.  **Wizard**: 初始化配置 DeepSeek/GLM 时，能自动列出模型供选择。
4.  **Ollama**: `/model` 能列出本地安装的所有模型。
5.  **错误处理**: 当 API 失败时，有优雅的降级处理 (手动输入)。
