# F11 初始化向导 (Init Wizard) 实现计划 - [DONE]

## 1. 目标 (Objective)
为首次使用 CodeAgent 的用户提供流畅的“开箱即用”体验。当检测到未配置模型 Provider 时，自动启动交互式向导，引导用户完成 API Key 配置或连接本地模型，并生成 `.env` 配置文件。

## 2. 触发机制
在 `src/index.ts` 启动流程中：
1.  调用 `registerProvidersFromEnv(engine)`。
2.  若返回的 `registered` 列表为空（即无有效 Provider）：
    *   **拦截退出**：不直接报错 `process.exit(1)`。
    *   **启动向导**：调用 `runInitWizard()`。

## 3. 交互流程设计

### Step 1: 欢迎与 Provider 选择
展示欢迎界面，并询问用户选择哪种模型服务。重点推荐代码能力强的模型。

```text
  ▝▜▄    CodeAgent CLI v1.0.0
    ▝▜▄  
   ▗▟▀   [!] 未检测到可用的大模型配置
  ▝▀     

? 请选择您要使用的模型服务商 (推荐代码能力强的模型):
❯ 🐋 DeepSeek (深度求索)   [推荐: deepseek-chat]
  🤖 GLM (智谱 AI)        [推荐: glm-4]
  🧠 Anthropic (Claude)   [推荐: claude-3-5-sonnet]
  ⚙️  OpenAI (官方/代理)    [需自备 Key/URL]
  🏠 Ollama (本地运行)     [免费/隐私]
```

### Step 2: 参数配置 (按 Provider 类型)

#### Case A: 主流厂商 (DeepSeek / GLM / Anthropic)
对于标准厂商，URL 和 Model Name 预设为最佳代码模型，仅需用户输入 API Key。

*   **DeepSeek**:
    *   Default URL: `https://api.deepseek.com/v1/chat/completions`
    *   Default Model: `deepseek-chat`
*   **GLM**:
    *   Default URL: `https://open.bigmodel.cn/api/paas/v4/chat/completions`
    *   Default Model: `glm-4`
*   **Anthropic**:
    *   Default URL: `https://api.anthropic.com/v1/messages`
    *   Default Model: `claude-3-5-sonnet-latest`

**交互**:
1.  Prompt: `请输入您的 API Key (sk-...):` (Masked Input)
2.  Validation: 尝试发送 "Hi" 验证连通性。

#### Case B: OpenAI / 自定义 (Custom)
允许用户完全自定义，兼容 OpenAI 协议的代理服务。

**交互**:
1.  Prompt: `API Base URL (默认: https://api.openai.com/v1):`
2.  Prompt: `Model Name (默认: gpt-4o):`
3.  Prompt: `API Key (sk-...):`
4.  Validation: 尝试发送 "Hi" 验证连通性。

#### Case C: Ollama (本地)
优先尝试连接本地默认端口。

**交互**:
1.  Action: 尝试连接 `http://localhost:11434/api/tags`。
2.  **若连接成功**:
    *   列出所有模型。
    *   Prompt: `请选择模型:` (Select)
3.  **若连接失败**:
    *   Prompt: `未检测到本地 Ollama。请输入远程 URL (或按 Enter 跳过):`

### Step 3: 生成配置文件 (.env)
1.  **读取模板**: 读取 `.env.example` 获取注释和结构。
2.  **填充数据**:
    *   将用户输入的 Key/URL/Model 填入对应 `_API_KEY`, `_API_URL`, `_MODEL` 字段。
    *   设置 `DEFAULT_PROVIDER=<selected_provider>`。
3.  **写入文件**: 生成 `.env` 文件。

### Step 4: 完成与启动
*   输出: `✅ 配置已保存至 .env！正在启动...`
*   自动进入 CLI 主界面 (`controller.start()`)。

## 4. 技术方案

### 4.1 新增模块
*   `src/cli/setup_wizard.ts`:
    *   `runInitWizard()`: 主入口。
    *   `askProvider()`: 选择 Provider。
    *   `askConfig(provider)`: 获取配置。
    *   `verifyConnection(config)`: 验证。
    *   `writeEnv(config)`: 写入 `.env`。

### 4.2 依赖库
*   `@inquirer/prompts`: 用于 `select`, `input`, `password` 交互组件。
*   `dotenv`: 解析现有 `.env`。

### 4.3 默认代码模型映射表 (Code Model Defaults)
| Provider | Default Model | 理由 |
| :--- | :--- | :--- |
| **DeepSeek** | `deepseek-chat` | 性价比高，代码能力强。 |
| **GLM** | `glm-4` | 国内稳定，长上下文。 |
| **Anthropic** | `claude-3-5-sonnet-latest` | 编程能力 SOTA。 |
| **OpenAI** | `gpt-4o` | 综合能力强。 |
| **Ollama** | `qwen2.5-coder` / `deepseek-coder` | 本地部署推荐。 |

## 5. 验收标准
1.  **无配置启动**: 删除 `.env` 后启动 CLI，应自动进入向导。
2.  **主流厂商配置**: 选择 DeepSeek/GLM/Anthropic，仅需输入 Key 即可成功配置并启动。
3.  **自定义配置**: 选择 OpenAI/Custom，能手动输入 URL/Model/Key 并成功配置。
4.  **Ollama**: 能自动检测本地模型列表并选择。
5.  **配置持久化**: 向导结束后，`.env` 文件生成正确，且包含 `DEFAULT_PROVIDER`。
6.  **验证失败处理**: 若 Key 无效，应提示重试或跳过。
