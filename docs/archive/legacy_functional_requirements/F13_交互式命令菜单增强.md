# Feature 实现文档：F13+ - 交互式 Slash 命令增强 (Enhanced Slash Interaction)

## 1. 背景与目标
为了提升 CLI 的专业感与操作效率，对 Slash 命令的交互逻辑进行了深度重构。目标是实现一套类似现代 IDE 命令面板（Command Palette）的极简、灵动且高度解耦的交互系统。

## 2. 核心功能实现

### 2.1 交互式实时建议 (Interactive Live Hints)
- **视觉风格**：采用极简主义设计，移除所有分支字符（如 `╰─`），仅通过**纯缩进**与**青色 (Cyan)** 颜色层次来标识。
- **键盘导航**：
    - 使用 `Up/Down` 方向键在建议列表中实时切换。
    - **高亮机制**：选中项以“粗体青色”显示命令名，“纯白色”显示描述；非选中项使用“暗青色”与“灰色”进行视觉弱化。
- **对齐优化**：建议列表与 Prompt 提示符 `❯ ` 后的输入起始点完美垂直对齐。

### 2.2 外科手术式提交重写 (Surgical Line Expansion)
- **功能描述**：当用户通过回车执行命令时，系统将提交的“部分匹配文本”原地替换为“完整命令名”。
- **原子化同步**：
    - 在清除建议列表之前，物理光标强制拉回输入行。
    - 使用 `\x1b[1F` (回到上一行) 配合 `\x1b[2K` (清行) 执行瞬间重写。
    - 同时同步 `readline` 内部的 `history` 和 `line` 状态，确保历史回溯也为完整指令。

### 2.3 极简侧边指示线 (Minimalist Sidebar Accent)
- **功能**：将 Slash 命令的执行结果（如 `/help`）与普通对话流在视觉上剥离，但避免沉重的边框。
- **设计**：
    - 移除全封闭方框，仅在输出内容的**左侧**增加一根粗细适中的**青色 (Cyan) 垂直竖线 `┃`**。
    - 垂直线不仅提供了明确的内容边界，还保持了终端流的连续性，不会阻断阅读感。

### 2.4 屏幕所有权管理 (Atomic Handoff)
- **独占模式**：引入 `ExclusiveMode`。在命令执行期间，严格抑制所有后台 UI 渲染（Bubbles、HUD 刷新），防止输出与交互界面发生重叠。

### 2.5 超简主义视觉体验 (Ultra-Minimalist Header)
- **输入框去噪**：完全移除了输入框上方冗余的“模型名 | 目录 | 帮助”黄色提示线。
- **核心交互**：界面回归极致简洁，仅保留 `❯ ` 提示符与底部的 HUD 状态栏。所有的全局状态（模型、Provider、Token 等）统一交由底部的 HUD 实时显示。

## 3. 技术架构
- **核心组件**：
    - `SlashHintManager`：负责状态化选择逻辑与极简渲染。
    - `StableFooterRenderer`：基于 ANSI 相对位移计算的非破坏性渲染引擎。
    - `REPL.executeCommand`：协调命令周期内的屏幕所有权移交。
    - `SlashCommands.getBestMatch`：独立于执行流的命令匹配逻辑，用于驱动 UI 展开。
