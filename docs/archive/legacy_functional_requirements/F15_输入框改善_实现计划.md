# F15 输入框视觉效果改善实现计划 (F15_输入框改善_实现计划)

## 1. 目标 (Objective)
提升 CLI 输入区域的视觉质感和信息密度，通过引入“双行提示符”和“视觉框架”，使 CodeAgent 的交互区域更像一个专业的 IDE 终端，而不仅是原始的命令行。

## 2. 视觉设计改进

### 2.1 结构化提示符 (Structured Prompt)
将当前的单行提示符 `CodeAgent > ` 改为双行结构：
*   **第一行 (Context Line)**: 显示当前活跃的模型、模式及路径信息。
*   **第二行 (Input Line)**: 精简的输入指示符。

**效果预览**:
```text
 ╭─ 🤖 glm-4-flash | 📁 CodeAgent | 💡 输入 / 获取帮助
 ╰─> /model
```

### 2.2 状态色彩同步
*   **空闲 (IDLE)**: 提示符前缀使用 `cyan` 或 `green`。
*   **多行捕获 (CAPTURE)**: 提示符颜色变为 `yellow`，并在 Context Line 显著标识 `[录制中 - 输入 EOF 结束]`。
*   **执行中 (THINKING)**: 隐藏提示符或显示动态加载图标。

### 2.3 边界与分隔
*   在上一条回复结束与新的输入框开始之间，自动绘制一条淡色分隔线 (`chalk.dim('─'.repeat(width))`)，明确区分对话历史。

## 3. 技术方案

### 3.1 `src/cli/terminal_manager.ts` (布局扩展)
*   新增 `renderInputHeader()` 方法，根据 `hud` 状态生成 Context Line。
*   支持获取当前终端宽度 (`process.stdout.columns`) 以绘制自适应的分隔线。

### 3.2 `src/cli/repl.ts` (提示符重构)
*   重写 `refreshPrompt()` 逻辑。
*   利用 `rl.setPrompt()` 设置多行字符串（注意：Node.js `readline` 对多行提示符的支持有限，可能需要将第一行通过 `console.log` 输出，第二行作为标准 `prompt`）。

### 3.3 交互体验优化
*   当用户切换 Provider 或 Model 后，立即触发 `refreshPrompt()` 并重新渲染输入区域，确保信息同步。

## 4. 实施阶段

### 第一阶段：分隔线与 Context Line
*   实实现每次用户输入开始前，打印一条分隔线。
*   输出淡色背景或边框风格的 Context 信息。

### 第二阶段：捕获模式视觉增强
*   当进入 `<<EOF` 模式时，改变终端左侧边框颜色，提示用户当前处于录制状态。

### 第三阶段：细节微调
*   处理窗口缩放时的渲染重绘。
*   优化在非 TTY 模式下的降级显示。

## 5. 验收标准
1.  启动后输入区域有清晰的 Context 标识（模型名）。
2.  输入历史与当前输入框之间有视觉分隔。
3.  在多行输入模式下，有明确的录制状态提示。
4.  `/model` 切换后，输入框上方的模型名称即时更新。
