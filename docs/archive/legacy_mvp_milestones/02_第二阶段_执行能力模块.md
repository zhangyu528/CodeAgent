## 第二阶段：执行能力模块 (Agent Loop 完整版)

在第一阶段跑通 LLM 与工具调用的基础上，第二阶段的目标是赋予 Agent 对物理世界产生实质影响的能力，并具备基本的复杂任务规划分解能力。此阶段核心开发对象对应 **P1 执行能力模块**。

### 包含的开发模块 (对应 P1 开发优先级)

- **Execution Environment (执行环境)**
  - **(已实现)** 建立了稳定的底层命令执行层 (`RunCommandTool`) 与全量文件写工具 (`WriteFileTool`)。
  - **(已实现)** **智能输出截断**：自动处理长日志输出，保护 LLM 上下文。
  - **(已实现)** **增强工具集**：新增 `ListDirectory` (结构化列目录)、`FileSearch` (代码搜索) 和 `ReplaceContent` (局部替换)。
  - **(已实现)** **路径安全加固**：所有文件操作工具均已实装工作区路径校验拦截。

- **Planner (任务规划器)**
  - **(已实现)** 引入任务分解能力 (`Planner` 类) 与多步骤自动化逻辑。
  - **(已实现)** **上下文连续性**：实现了跨步骤的对话历史继承，使 Agent 在执行连贯任务时具备长期记忆能力。
  - **(已实现)** **失败重规划**：步骤失败时自动触发动态规避与重新规划。

- **Self-Correction 闭环**
  - **(已实现)** 建立了“修改-运行-报错分析-再修改”的初步自愈链路，使 Agent 能够根据 `stderr` 反馈纠正操作。

### 系统能力与验收标准

系统在此阶段应具备以下增强能力：

**User Task → LLM → Plan → Tool (Execution) → Result**

可以通过编写一个中等复杂度的测试入口指示 Agent 完成连贯性任务（例如：“跑通项目单元测试，如果报错帮我分析并修复第一个错误”），系统能够：
1. 分析指令，列出明显的步骤 (1. 运行测试 2. 获取输出分析 3. 可能的修改操作)。
2. 通过 Execution Environment 实际运行终端中的 `npm test` 或相近命令。
3. 读入测试结果后，若存在失败情况，调用 Controller 执行文件分析与修改工具。
  4. 提供连贯的长步骤体验，证明 Agent 具备“做事并拿到结果”的能力。
