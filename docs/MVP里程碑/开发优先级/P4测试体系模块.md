# P4：自动化测试体系模块实现文档 (Node.js + TypeScript)

为了保证 CodeAgent 在复杂项目场景下的长期稳定性与逻辑准确性，P4 阶段建立了一套完整的自动化测试体系。该体系从单元逻辑到全链路工作流进行了层级化覆盖。

> [!NOTE]
> **当前状态**：P4 测试体系模块已于 2026-03-11 完成 100% 开发与目录重构。

## 1. 测试层级架构 (Testing Hierarchy)

测试体系被划分为四个核心象限，确保从原子代码到系统行为均受监控：

| 层级 | 目录 | 重点对象 | 运行特性 |
| :--- | :--- | :--- | :--- |
| **Unit (单元)** | `src/tests/unit/` | `MemoryManager`, `SecurityLayer` | **极速**：无需 LLM API，纯逻辑验证。 |
| **Security (安全)** | `src/tests/security/` | `PathGuard` | **专项**：验证越权拦截与攻击绕过能力。 |
| **Integration (集成)** | `src/tests/integration/` | `AgentController`, `Planner` | **链路**：验证组件间协作与工具调用逻辑。 |
| **E2E (端到端)** | `src/tests/e2e/` | `ComprehensiveWorkflow` | **场景**：验证复杂多步任务的闭环解决能力。 |

## 2. 核心补强特性

### 2.1 记忆稳定性验证 (`MemoryManager`)
- **角色轮替锁**：单元测试确保 `truncate` 逻辑不会导致 `user` 消息缺失。
- **系统提示词持久化**：验证在高频截断下系统身份设定的绝对留存。

### 2.2 防越权专项 (`SecurityLayer`)
- **路径逃逸拦截**：针对 `../` 及其变体的路径计算逻辑进行 100% 覆盖测试。
- **高危指令黑名单**：模拟执行破坏性 Shell 命令并验证 Agent 的拒绝响应。

### 2.3 健壮性参数支持
- **Max Iterations 可配置**：在 `AgentController` 中引入配置项，防止复杂任务导致的死循环。

## 3. 运行指南

我们在 `package.json` 中整合了快捷入口，支持分级运行：

```bash
# 执行全部回归测试
npm test

# 仅执行本地单元测试 (CI/CD 推荐)
npm run test:unit

# 执行安全性专项
npm run test:security
```
