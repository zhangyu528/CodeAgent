## 1. Observability (可观测性与视觉增强)

为了让用户清晰了解 Agent 的后台逻辑，系统引入了分级与彩色的实时反馈机制。

### 1.1 核心特性
- **动态加载状态 (Spinners)**：集成 `ora` 库，在 Agent 处于“思考中”或“执行工具中”时显示动态加载动画。
- **语义化色彩 (Chalk Styling)**：使用 `chalk` 对输出进行分色：
  - **Thought (灰色)**：展示 Agent 的推理逻辑。
  - **Action (黄色)**：明确指示正在调用的工具名。
  - **Observation (品红)**：展示工具产生的反馈结果。
  - **Answer (绿色)**：醒目标注最终交付的答案。
- **成本与 Token 监控 (Telemetry)**：
  - 实时显示当前上下文占用的 Tokens。
  - 累计统计整个会话的 Tokens 消耗并预估 API 费用（单位：$）。

### 1.2 关键类与方法 (`src/utils/logger.ts`)
```typescript
export class Logger {
  // 开启/停止动态加载动画
  startSpinner(text: string): void;
  stopSpinner(): void;
  
  // 打印不同阶段的彩色日志
  thought(text: string): void;
  action(name: string, args: any): void;
  tokenUsage(usage: number, telemetry?: TelemetryMonitor): void;
}
```

## 2. CLI Interface (现代化命令行交互)

CLI 层作为用户交互的唯一入口，现在提供了更加标准和稳健的操作模式。

### 2.1 核心特性
- **唯一交互式入口 (Unified REPL)**：程序启动后直接进入 `CodeAgent >` 提示符，支持连续对话与上下文记忆。
- **全局命令支持**：通过 `package.json` 的 `bin` 字段将该工具映射为 `codeagent` 指令，支持全局运行。
- **现代化 HITL 交互**：使用 `@inquirer/prompts` 提供了原生的终端单选/多选确认界面。

### 2.2 启动方式
- 直接执行 `npm start` 或全局命令 `codeagent`。
