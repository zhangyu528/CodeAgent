# CLI 可上线版本必备需求

本文件记录可上线版本 CLI 的必备能力与理由，作为后续实现的验收依据。

## 必备能力（核心功能）

- `run`：单次任务执行入口【P0】
- `chat`：多轮交互模式（支持退出）【P2】
- `config`：配置读写（get/set/list）【P0】
- `--help` / `--version`：基础可用性支持【P1】

## 安全与风险控制

- **HITL 交互挂载到 SecurityLayer**【P0】
  - SecurityLayer 负责是否需要确认
  - CLI 负责展示 y/n 并回传结果
  - 单一 HITL 策略，多入口共享

## 稳定性与可维护性

- **错误可读输出 + 非 0 exit code**【P0】
  - 便于脚本化与自动化判断
- **配置来源提示（env / config / CLI）**【P1】
  - 防止“改了没生效”的困惑
  - 避免隐式覆盖导致的模型/参数偏差
  - 支持多环境切换与排错

## 日志与可观测

- 分级日志（debug/info/warn/error）【P1】
- 可选日志落盘【P2】

## 交互体验（可上线基础）

- 清晰的退出机制（`exit` / `quit` / Ctrl+C）【P1】
- 最小诊断命令（可选 `doctor`）【P3】
